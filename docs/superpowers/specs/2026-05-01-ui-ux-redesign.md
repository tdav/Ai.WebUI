## 1. Сайдбар (NavMenu.razor)

### Проблемы (исправляем)
1. Кнопка «+ Проект» ведёт на `/` вместо создания проекта
2. Нет кнопки создания чата в сайдбаре
3. «Выйти» — голый текст без иконки
4. Свёрнутый sidebar полностью пуст

### Решение

**Кнопка «+ Проект»**  
Открывает модальное окно `CreateProjectModal` — не перенаправляет на `/`.

**Создание чата из сайдбара**  
Внутри раскрытого проекта — в конце списка чатов — кнопка «+ чат» (вариант C).  
Рендерится только для активного (раскрытого) проекта.

**«Выйти»**  
Иконка выхода (`→` или `⎋`) + текст «Выйти».

**Свёрнутый sidebar**  
Иконки проектов: первая буква имени проекта в кружке. При наведении — tooltip с полным именем.

**Обработка ошибок**  
Все действия (создание проекта, создания чата) при ошибке показывают уведомление. Состояние сайдбара не ломается.

## 2. Создание проекта (`CreateProjectModal.razor`)

### Новый компонент
Модальное окно с полями:
- **Название** — обязательное, `string`, max 100 символов
- **Описание** — необязательное, `string?`, max 500 символов

### Флоу
1. Клик «+ Проект» → модалка открывается
2. Пользователь вводит название (Enter или кнопка «Создать»)
3. Создаётся `Project`
4. Автоматически создаётся первый `Chat` (Title = «Новый чат»)
5. Закрытие модалки → `NavigateTo("/chat/{chatId}")`
6. Сайдбар обновляет список проектов

### Обработка ошибок
- Пустое название → inline-ошибка под полем, не закрываем модалку
- Ошибка БД → сообщение внутри модалки, введённые данные сохраняются
- Кнопка «Создать» блокируется во время запроса (`isLoading`)

## 3. Создание чата из сайдбара

### Флоу
1. Клик «+ чат» в конце списка чатов раскрытого проекта
2. Немедленно создаётся `Chat` (Title = «Новый чат», ProjectId = текущий проект)
3. `NavigateTo("/chat/{chatId}")`

### Обработка ошибок
- Ошибка БД → toast-уведомление, навигация не происходит

## 4. Прикрепление файлов (ChatInput.razor)

### Что меняется
`DocumentUpload.razor` удаляется. Его логика (вызов `DocService.ExtractTextAsync`, сохранение `Document` в БД) переезжает напрямую в `ChatInput.razor`.

### Новый ChatInput
**Строка ввода:**
```
[📎] [textarea...                    ] [↑]
```

**Chips над полем** (появляются после прикрепления):
```
[📄 report.pdf  12кб  ×] [📝 notes.md  4кб  ×]
[📎] [textarea...                    ] [↑]
```

### Поведение
- Клик 📎 → открывает `<InputFile>` (форматы: .pdf, .txt, .md, .docx, .xlsx, .pptx)
- Drag-and-drop на область чата → тоже работает
- Файлы хранятся в локальном состоянии до отправки

### Пайплайн (логика не меняется, переносится)
1. Выбор файла → `DocService.ExtractTextAsync(file)` → `ExtractedText`
2. `Document` сохраняется в БД (`ChatId`, `UserId`, `ExtractedText`)
3. При отправке сообщения `ExtractedText` включается в контекст через `chatDocs`

### Обработка ошибок
- Неподдерживаемый формат → ошибка рядом с chips (не крашит компонент)
- Файл слишком большой → то же
- Ошибка извлечения текста → chip помечается ошибкой, файл не отправляется

### EventCallback
`ChatInput` получает новый параметр:
```csharp
[Parameter] public EventCallback<List<Document>> OnDocumentsChanged { get; set; }
```
`ChatPage.razor` подписывается на него вместо `DocumentUpload`.
